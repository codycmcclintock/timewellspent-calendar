“Cannot connect using SSL” then “unable to verify account”?

You do not need a username or password for this calendar (leave them blank). When iOS asks about SSL, do not choose “Continue” without SSL — that breaks the link. Tap Cancel, then use the jsDelivr URL below (same calendar, different server). In the field labeled Server, paste the entire https://…calendar.ics URL even though the label says “Server” — that is normal for Apple.

If the screen title says CalDAV or CardDAV, go back — that is the wrong type of account. You want Subscribed Calendar or Subscription Calendar only.

Easiest path (Apple’s current guide)

Open the Calendar app → tap Calendars (bottom center) → Add Calendar → Add Subscription Calendar → paste one https:// URL from below → Find / Subscribe.

Settings path (older wording)

Settings → Calendar → Accounts → Add Calendar → Add Subscribed Calendar. Paste a full https://…calendar.ics URL into Server. Leave username and password empty. Keep SSL on.
